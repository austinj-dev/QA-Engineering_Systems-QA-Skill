# QA Engineering / Systems QA

**Process and Skill Developed by Austin Johnnic**

Designing rigorous test plans, validation pipelines, reliability checks, and production readiness verification.

**Motto: "Never trust — always verify. Never assume coverage — always test deeper."**

## What This Is

A comprehensive QA engineering skill that produces testing artifacts and can directly execute tests. Supports full-repo testing, feature-specific testing, and targeted aspect testing across any language, any stack, any project type.

## Features

- **37 testing phases** (dynamically enabled based on codebase discovery)
- Two-mode architecture: Discovery → Fix (with strict approval gate)
- Full OWASP Top 10 2025 security testing (NEVER skippable)
- WCAG 2.2 AA accessibility testing
- API contract testing (schemas, status codes, auth per endpoint)
- Multi-tenant isolation testing
- State transition testing for every entity with statuses
- Session management & rate limiting verification
- Email, webhook, and real-time feature testing
- File upload/download & export/print testing
- Data volume & pagination testing (0 to 10K+ records)
- Error message quality testing
- Onboarding & first-run experience testing
- Cross-browser testing (Chrome, Firefox, Safari, Edge, mobile)
- Load & stress testing methodology
- CI/CD pipeline verification
- Backup & recovery verification
- Monitoring & observability readiness
- Compliance & legal testing (GDPR, CCPA, PCI)
- Documentation QA (README, API docs, help content accuracy)
- Visual regression testing (Playwright screenshot comparison)
- Concurrency & race condition testing
- Playwright test script generation
- AI feature testing (functional, safety, performance, cost)
- 15+ QA documents generated
- Production readiness confidence rating (0-100%)
- Tech-stack agnostic (works with any language/framework)
- Tool-agnostic (Claude Code, Cursor, Windsurf, chat)

> **#AllGapsMatter** — Every testing gap is a production incident waiting to happen.
